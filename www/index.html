Waarom een designer?

Je kunt natuurlijk een grafische interface maken met MAUI op basis van alleen code.
Je moet dan een inschatting maken hoe je grafische interface eruit gaat zien op basis van de code die je hebt geschreven. Een andere mogelijkheid is om het programma te compileren en te zien hoe het eruit ziet, maar dat kan wel even duren. Daarom is de MAUI Designer een oplossing. Je selecteert een XAML bestand en de MAUI Designer kijkt ieder vijf seconden naar wijzigingen. Als de XAML gewijzigd is, dan toont de MAUI Designer hoe je applicatie eruit zou komen te zien als je het gewijzigde XAML bestand zou gebruiken.

Workflow

Een handige manier van werken is heel persoonlijk. Wat je bijvoorbeeld kunt doen is in het begin een XAML bestand laden. Terwijl je code schrijft, kan je met Alt+Tab voor Windows en Cmd+Tab voor OS X wisselen tussen vensters. Op die manier kan je snel even kijken hoe je applicatie eruit gaat zien. Daarna kan je weer snel in de code duiken om verbeteringen aan te brengen.

Bijdragen

Als je een foutje in het programma ziet, of je hebt een suggestie hoe het verbeterd kan worden, dan kan je een issue of een Pull Request openen op onze repository.
